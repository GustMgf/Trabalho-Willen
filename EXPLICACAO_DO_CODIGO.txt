EXPLICACAO DO CODIGO - PROJETO BOOKSWAP
========================================

Este arquivo explica a estrutura atual do projeto BookSwap, o objetivo de cada
pasta e o papel dos arquivos principais.


1. VISAO GERAL DO PROJETO
-------------------------

O BookSwap e uma aplicacao web para troca de livros entre usuarios.

O projeto tem duas partes principais:

- Frontend: arquivos HTML, CSS e JavaScript dentro da pasta public.
- Backend/API: servidor Node.js com Express dentro da pasta src.

O servidor Express tambem entrega os arquivos do frontend. Por isso, ao rodar:

  npm start

o site e a API ficam no mesmo endereco:

  http://localhost:3000

A API fica em:

  http://localhost:3000/api

O banco de dados usado atualmente e MySQL.


2. ARQUIVOS DA RAIZ
-------------------

README.md

Arquivo de documentacao principal do projeto. Explica como instalar, criar o
banco, rodar o servidor e testar as rotas principais.


database.sql

Script SQL para criar o banco bookswap e suas tabelas.

Tabelas principais:

- usuarios: guarda nome, email, senha, cidade, estado e foto_perfil.
- livros: guarda livros cadastrados para troca.
- livros_desejados: guarda a lista de desejos dos usuarios.
- combinacoes: estrutura para sugestoes/matches entre desejo e livro.
- trocas: guarda propostas de troca.
- mensagens: guarda mensagens vinculadas a uma troca.

Observacao: a coluna foto_perfil guarda o caminho da imagem, nao a imagem
inteira. A imagem fica salva em pasta dentro de public/img/uploads/perfis.


package.json

Define as dependencias e scripts do projeto.

Scripts:

- npm start: roda src/server.js.
- npm run dev: roda com nodemon, reiniciando automaticamente em alteracoes.

Dependencias:

- express: cria o servidor e as rotas HTTP.
- mysql2: conecta o Node.js ao MySQL.
- nodemon: dependencia de desenvolvimento para reiniciar o servidor.


package-lock.json

Arquivo gerado pelo npm. Ele registra as versoes exatas das dependencias
instaladas. Normalmente nao e editado manualmente.


node_modules/

Pasta criada pelo npm install. Contem as bibliotecas instaladas.

Nao deve ser editada manualmente.


tmp/

Pasta usada para logs temporarios do servidor.

Arquivos atuais:

- server.out.log
- server.err.log

Nao faz parte da regra de negocio do sistema. Serve apenas para saida/logs.


3. PASTA public/
----------------

A pasta public contem o frontend do BookSwap. Tudo que esta aqui e servido
como arquivo estatico pelo Express.

Exemplo:

  public/books.html

fica acessivel em:

  http://localhost:3000/books.html


3.1 Arquivos HTML
-----------------

public/index.html

Pagina inicial/apresentacao do BookSwap.
Mostra a proposta do sistema e links para login/cadastro.
Tem data-no-shell="true", entao nao usa o header global gerado pelo JS.


public/login.html

Tela de login.
Usa public/js/auth.js para enviar email e senha para a API.
Quando o login funciona, salva o usuario no localStorage.


public/register.html

Tela de cadastro.
Usa public/js/auth.js para criar usuario pela API.


public/books.html

Tela principal de livros disponiveis.
Mostra os livros cadastrados na tabela livros.
Usa public/js/books.js para carregar e renderizar os cards.
Atualmente a grade usa 3 livros por linha no desktop.


public/explore.html

Tela de exploracao/busca no acervo.
Mostra livros e permite filtrar por titulo, autor, genero ou condicao.


public/add-book.html

Tela para cadastrar um livro.
Permite enviar capa do livro.
O frontend envia a imagem em base64 e o backend salva o arquivo em
public/img/uploads.


public/my-books.html

Tela do acervo do usuario logado.
Mostra apenas os livros cadastrados pelo usuario atual.
Permite editar e excluir livros.


public/wanted-books.html

Tela de livros desejados.
Permite adicionar desejos, editar desejos e excluir desejos.
Os desejos nao usam imagem. Sao exibidos como cards textuais com genero,
titulo, autor e acoes.


public/matches.html

Tela de combinacoes inteligentes.
Mostra sugestoes baseadas nos livros desejados do usuario e nos livros
disponiveis de outras pessoas.
Atualmente exibe 3 cards por linha no desktop.


public/trades.html

Tela de trocas.
Mostra propostas de troca envolvendo o usuario logado.
Permite alterar status das trocas, conforme a regra do backend.


public/chat.html

Tela de conversa/mensagens vinculadas a uma troca.
Usa public/js/chat.js.


public/profile.html

Tela de perfil do usuario.
Mostra dados pessoais, estatisticas, foto de perfil, alteracao de senha,
remocao de foto e exclusao da conta.

As estatisticas de Livros, Desejos e Trocas nao ficam salvas em uma coluna
separada. Elas sao calculadas no frontend contando registros das tabelas:

- livros
- livros_desejados
- trocas

5. public/js/
-------------

Esta pasta contem os JavaScripts do frontend.


public/js/api.js

Arquivo global mais importante do frontend.

Responsabilidades:

- define API_BASE_URL como http://localhost:3000/api;
- cria a funcao apiRequest para chamar a API;
- salva e le o usuario logado no localStorage;
- limpa sessao no logout;
- gera o shell compartilhado do site:
  - header;
  - links superiores;
  - menu hamburguer;
  - dropdown de perfil;
  - navegacao inferior;
- contem helpers usados por outras paginas:
  - escapeHtml;
  - formatStatus;
  - formatDate;
  - getBookCover;
  - bookCardTemplate;
  - proposeTradeForBook;
  - showToast.

Logout:

A funcao logout limpa localStorage e sessionStorage para chaves como:

- bookswap:user
- bookswap:token
- token
- auth
- jwt
- currentUser
- user

Depois redireciona para login.html ou para outro destino passado por parametro.


public/js/auth.js

Controla login e cadastro.

No login:

- envia email e senha para /api/autenticacao/entrar;
- recebe o usuario;
- salva com BookSwap.setCurrentUser;
- redireciona para books.html.

No cadastro:

- envia nome, email, senha, cidade e estado;
- redireciona para login.html apos sucesso.


public/js/books.js

Controla varias telas ligadas a livros:

- books.html;
- explore.html;
- my-books.html;
- add-book.html;
- wanted-books.html.

Responsabilidades:

- listar livros;
- filtrar livros;
- cadastrar livro;
- editar livro;
- excluir livro;
- ler imagem da capa;
- listar livros desejados;
- adicionar desejo;
- editar desejo;
- excluir desejo.

Observacao:

Apesar do nome books.js, ele tambem controla a tela de desejos.


public/js/profile.js

Controla a pagina profile.html.

Responsabilidades:

- carregar dados do usuario logado;
- preencher resumo e formulario;
- exibir foto de perfil;
- escolher nova foto;
- remover foto;
- salvar perfil;
- alterar senha;
- excluir conta;
- sair da conta.

Foto de perfil:

O frontend le o arquivo como base64 apenas para enviar para a API. O backend
salva a imagem em pasta e devolve o caminho.


public/js/matches.js

Controla matches.html.

Responsabilidades:

- carregar combinacoes da API;
- filtrar combinacoes do usuario logado;
- montar cards de sugestao;
- permitir propor troca a partir de uma combinacao.


public/js/trades.js

Controla trades.html.

Responsabilidades:

- listar trocas do usuario;
- mostrar livro solicitado e parceiro;
- atualizar status da troca.


public/js/chat.js

Controla chat.html.

Responsabilidades:

- localizar troca ativa do usuario;
- listar mensagens;
- enviar mensagens;
- exibir resumo da conversa.

9. src/controllers/
-------------------

Controllers sao os arquivos que executam a regra de negocio quando uma rota e
chamada.


src/controllers/authController.js

Controla cadastro e login.

Funcoes principais:

- register: cria usuario.
- login: valida email/senha e retorna usuario.

Observacao:

A senha atualmente e comparada diretamente como texto. Nao ha hash de senha.
Em um sistema real seria recomendado usar bcrypt ou similar.


src/controllers/userController.js

Controla usuarios.

Funcoes principais:

- listar usuarios;
- buscar usuario por id;
- atualizar usuario;
- excluir usuario.

Tambem controla foto de perfil:

- recebe base64;
- valida PNG/JPG/WEBP;
- salva arquivo em public/img/uploads/perfis;
- remove foto antiga;
- salva no banco apenas o caminho.


src/controllers/bookController.js

Controla livros cadastrados.

Funcoes principais:

- listar livros;
- buscar livro por id;
- criar livro;
- atualizar livro;
- excluir livro.

Tambem controla capas de livro:

- recebe imagem;
- valida tipo;
- salva em public/img/uploads;
- guarda caminho em capa_imagem.


src/controllers/wantedBookController.js

Controla livros desejados.

Funcoes principais:

- listar desejos;
- buscar desejo por id;
- criar desejo;
- atualizar desejo;
- excluir desejo.

Os desejos nao possuem imagem.


src/controllers/matchController.js

Controla combinacoes.

Ele cruza livros desejados com livros cadastrados por outros usuarios.
Usa o algoritmo de match em src/utils/matchAlgorithm.js.


src/controllers/tradeController.js

Controla trocas.

Funcoes principais:

- listar trocas;
- buscar troca por id;
- criar troca;
- atualizar status;
- excluir troca.

Tambem valida se os livros pertencem aos usuarios corretos e atualiza status
dos livros quando necessario.


src/controllers/messageController.js

Controla mensagens de uma troca.

Funcoes principais:

- listar mensagens de uma troca;
- enviar mensagem.

12. src/utils/
--------------

src/utils/mapeadoresBanco.js

Converte nomes de colunas do MySQL para nomes usados no frontend/API.

Exemplo:

- usuario_id vira usuarioId e userId.
- capa_imagem vira capaImagem e coverImage.
- foto_perfil vira fotoPerfil, imagemPerfil e profilePhoto.

Isso permite compatibilidade entre nomes em portugues e ingles.


src/utils/matchAlgorithm.js

Define a pontuacao das combinacoes.

Compara dados como:

- titulo;
- autor;
- genero.

Retorna pontuacao para indicar compatibilidade entre um livro desejado e um
livro encontrado.


src/utils/fileHandler.js

Arquivo legado.

Ele tem funcoes parecidas com manipulacao de JSON, mas atualmente a API usa
MySQL. O proprio arquivo indica:

  Persistencia em JSON desativada. Use os controllers com MySQL.

Ou seja: existe no projeto, mas nao e usado como persistencia atual.

14. FLUXO DE LOGIN E SESSAO
---------------------------

1. Usuario entra em login.html.
2. auth.js envia email/senha para a API.
3. authController valida no banco.
4. O frontend salva o usuario em localStorage na chave:

   bookswap:user

5. Paginas protegidas usam BookSwap.requireAuth().
6. Se nao houver usuario salvo, redireciona para login.html.

Logout:

O logout limpa dados de localStorage e sessionStorage e redireciona para
login.html.


15. FLUXO DE CADASTRO DE LIVRO
------------------------------

1. Usuario acessa add-book.html.
2. Preenche titulo, autor, genero, condicao, descricao, status e capa.
3. books.js le a capa como base64.
4. Envia para POST /api/livros.
5. bookController salva os dados no MySQL.
6. Se tiver imagem, salva o arquivo em public/img/uploads.
7. O banco guarda apenas o caminho em capa_imagem.


16. FLUXO DE FOTO DE PERFIL
---------------------------

1. Usuario escolhe foto em profile.html.
2. profile.js le a imagem como base64.
3. Envia para PUT /api/usuarios/:id.
4. userController valida a imagem.
5. Salva arquivo em public/img/uploads/perfis.
6. Banco guarda caminho em foto_perfil.
7. O frontend usa esse caminho para exibir a imagem.


17. FLUXO DE DESEJOS
--------------------

1. Usuario acessa wanted-books.html.
2. Adiciona titulo, autor e genero.
3. books.js envia para POST /api/livros-desejados.
4. wantedBookController salva na tabela livros_desejados.
5. A lista e renderizada sem imagem, apenas com dados textuais.
6. Usuario pode editar ou excluir o desejo.


18. FLUXO DE COMBINACOES
------------------------

1. Usuario cadastra desejos.
2. A API compara esses desejos com livros cadastrados por outros usuarios.
3. matchAlgorithm calcula pontuacao.
4. matches.html mostra as combinacoes.
5. Usuario pode propor troca.


19. FLUXO DE TROCAS E MENSAGENS
-------------------------------

1. Usuario clica em propor troca.
2. A API cria registro na tabela trocas.
3. trades.html mostra as trocas do usuario.
4. chat.html permite conversar dentro de uma troca.
5. As mensagens ficam na tabela mensagens.


20. O QUE EXISTE MAS NAO ESTA SENDO USADO OU E LIMITADO
-------------------------------------------------------

src/data/*.json

Nao sao usados pela API atual. Eram persistencia antiga em JSON.


src/utils/fileHandler.js

Tambem e legado. As funcoes de escrita/leitura JSON foram desativadas.


Tabela combinacoes

Existe no database.sql, mas as combinacoes atuais tambem podem ser calculadas
dinamicamente pelo controller/algoritmo. Dependendo da implementacao final, a
tabela pode ser usada para persistir historico ou cache, mas o comportamento
principal vem do controller de combinacoes.


Autenticacao com token/JWT

Nao existe JWT real implementado atualmente.

O frontend limpa chaves como token, auth e jwt no logout por seguranca, mas o
login atual salva apenas o objeto do usuario em localStorage.


Senha com hash

Nao ha hash de senha implementado. As senhas sao salvas/comparadas diretamente.
Para producao, seria importante adicionar hash com bcrypt.


Testes automatizados

Nao ha pasta de testes nem scripts de teste configurados no package.json.


21. RESUMO FINAL
----------------

O BookSwap atual e uma aplicacao full-stack simples:

- Express serve API e frontend.
- MySQL guarda os dados.
- public contem as telas.
- public/js contem a logica do navegador.
- src/controllers contem a regra de negocio.
- src/routes define os endpoints.
- src/utils contem conversores e algoritmos auxiliares.

Os pontos mais importantes do sistema sao:

- usuarios podem cadastrar livros;
- usuarios podem cadastrar desejos;
- o sistema gera combinacoes;
- usuarios podem propor trocas;
- usuarios podem conversar por mensagens;
- usuarios podem editar perfil, foto e senha;
- usuarios podem sair ou excluir a conta.
